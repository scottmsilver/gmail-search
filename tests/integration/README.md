# Integration tests

These tests hit **real services** — claudebox, MCP tools server,
Postgres, gmail-search FastAPI — and **cost real Anthropic API
credits** per run.

They are gated behind `pytest -m integration` and skip cleanly if any
of the four services is unreachable.

## What they need

Four services up on the dev box:

| Service | Endpoint | Bring up with |
|---|---|---|
| gmail-search FastAPI | `http://127.0.0.1:8090` | `gmail-search serve --port 8090` |
| MCP tools server | `http://127.0.0.1:7878` | `python -m gmail_search.agents.mcp_tools_server` |
| claudebox | `http://127.0.0.1:8765` | `bash deploy/claudebox/start.sh` |
| Postgres (paradedb) | `127.0.0.1:5544` | `docker compose up -d pg` |

Tokens (read by `conftest.py`):

- `GMAIL_MCP_ADMIN_TOKEN` — env var, or fall back to
  `scripts/mcp_admin_token` (auto-generated by the MCP server when
  the env var is unset).
- `CLAUDEBOX_API_TOKEN` — env var, or fall back to
  `deploy/claudebox/.env`.

## Run them

```bash
# Default `pytest -q` SKIPS this directory entirely (the marker is
# excluded via addopts in pyproject.toml).
pytest -m integration -v

# Just the MCP side-channel tests:
pytest -m integration -v tests/integration/test_mcp_side_channel_e2e.py

# Collect-only sanity check (no API spend):
pytest -m integration -v --co
```

## Cleanup

The `scratch_cleanup` fixture removes `data/agent_scratch/itest-*/`
after each test. Registered MCP sessions are unregistered in
`finally` blocks. Failed test runs may leave a registered session in
the MCP server — restart the MCP server or DELETE
`/admin/sessions/<id>` to clear.
